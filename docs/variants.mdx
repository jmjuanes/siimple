# Variants

Variants can be used to register custom modifiers that are used to change the look and feel of components. 

```scss
@use "siimple" as siimple;

@include siimple.configure(
    $variants: (
        "button": (
            "default": (
                "background-color": "primary",
                "color": "white",
            ),
            "secondary": (
                "background-color": "secondary",
            ),
        ),
    ),
);
```

## Default variant

In the `default` field you can provide the styles that will be applied to the default component styles. For example, all styles defined in `$variants.button.default` will be applied directly to the `button` class: 

```scss
@use "siimple" as siimple;

@include siimple.configure(
    $variants: (
        "button": (
            "default": (
                "background-color": "primary",
                "color": "white",
            ),
        ),
    ),
);
```

Will generate: 

```css
.button {
    background-color: #33333;
    color: white;
    /* Other default button styles */
}
```

## Additional variants

Any other field will generate an additional class that can be only applied to the corresponding component, with the class `.is-{variant_name}`. For example, this variant:

```scss
@use "siimple" as siimple;

@include siimple.configure(
    $variants: (
        "button": (
            "secondary": (
                "background-color": #000000,
            ),
        ),
    ),
);
```

Will generate:

```css
.button.is-secondary {
    background-color: #000000;
}
```
