# Variants

Variants can be used to register custom modifiers that are used to change the look and feel of elements. 

```js title=siimple.config.js
export default {
    variants: {
        button: {
            default: {
                backgroundColor: "primary",
                color: "white",
            },
            secondary: {
                backgroundColor: "secondary",
            },
        },
    },
};
```

## Default variant

In the `default` field you can provide the styles that will be applied to the default component styles. For example, all styles defined in `variants.button.default` will be applied directly to the `button` class: 

```js title=siimple.config.js
export default {
    variants: {
        button: {
            default: {
                backgroundColor: "primary",
                color: "white",
            },
        },
    },
};
```

Will generate: 

```css
.button {
    background-color: #33333;
    color: white;
    /* Other default button styles */
}
```

## Additional variants

Any other field will generate an additional class that can be only applied to the corresponding component, with the class `.is-{variant_name}`. For example, this variant:

```js title=siimple.config.js
export default {
    variants: {
        button: {
            secondary: {
                backgroundColor: "#000000",
            },
        },
    },
};
```

Will generate:

```css
.button.is-secondary {
    background-color: #000000;
}
```
