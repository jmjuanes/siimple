# Configuration


**Siimple** is based in a set of configuration rules written in SCSS, where you can define your theme, the components that you need to include and additional plugins and styles.

This is an example of a configuration file for **siimple**: 

```scss
// index.scss 
@use "siimple" as siimple;

// Configure siimple
@include siimple.configure(
    $prefix: "",
    $scales: (
        "colors": (
            "primary": siimple.get-color("blue", 500),
            "secondary": siimple.get-color("mint", 600),
            "text": siimple.get-color("gray", 800),
            "background": #ffffff,
        ),
        "fonts": (
            "body": (Roboto, sans-serif),
            "heading": (Montserrat, sans-serif),
            "monospace": monospace,
        ),
    ),
    $flags: (
        "use-root": true,
        "use-borderbox": true,
    ),
);

// Generate siimple
@include siimple.css;
```

### Prefix

The `$prefix` section allows you to customize the prefix that will be used for naming the components of **siimple**. Using a prefix is useful to prevent naming collision with your custom classes or with other CSS libraries.

For example, you can use `si-` as prefix by setting the `$prefix` section:

```scss
@include siimple.configure(
    $prefix: "si-",
);
```

Now every component class will have the configured prefix:

```css
.si-btn {
    /* ... */
}
.si-btn.is-primary {
    /* ... */
}
.si-btn.is-secondary {
    /* ... */
}
```

### Breakpoints

You can customize the breakpoints used in **siimple** adding a map in the `$breakpoints` section with your custom breakpoints. The keys in this map are the breakpoints names, and the value can be one of the following:

- A screen size in `px`, that will be set as the `min-width` value.
- A map with the values for `min-width` and `max-width`. For example, `("min": 0px, "max": 600px)` will generate `@media screen and (min-width: 0px) and (max-width: 600px)`.

This is the default breakpoints configuration: 

```scss
@include siimple.configure(
    $breakpoints: (
        "mobile": ("min": 0px, "max": 600px),
        "tablet": ("min": 600px),
        "desktop": ("min": 1200px),
        "fullhd": ("min": 1800px),
    ),
);
```


### Scales

In the `$scales` section you can configure the list of CSS properties specific for your project.

```scss
@use "siimple" as siimple;

@include siimple.configure(
    $scales: (
        "colors": (
            "primary": siimple.get-color("blue", 500),
            "secondary": siimple.get-color("mint", 600),
            "text": siimple.get-color("gray", 800),
            "background": #ffffff,
        ),
        "fonts": (
            "body": (Roboto, sans-serif),
            "heading": (Montserrat, sans-serif),
            "monospace": monospace,
        ),
    ),
);
```

Visit the [scales guide](/scales) to read more about how to define your custom scales.


### Variants

You can use the `$variants` section to customize the style of the components provided by **siimple** or included using plugins. 

```scss
@use "siimple" as siimple;

@include siimple.configure(
    $variants: (
        "button": (
            "default": (
                "background-color": "primary",
                "color": "white",
            ),
            "secondary": (
                "background-color": "secondary",
            ),
        ),
    ),
);
```

Visit the [variants guide](/variants) to read more about how to customize your components using variants.


### Configuration flags

Use the `$flags` to enable or disable the modules that will be included when calling the `siimple.css` mixin.

| Flag | Description |
|------|-------------|
| `"use-reboot"` | Loads the reboot module. Default is `true`. |
| `"use-root"` | Add root styles to the `<html>` element. Default is `true`. |
| `"use-borderbox"` | Adds a global `box-sizing: border-box`. Default is `true`. |
| `"use-markup"` | Loads the markup module. Default is `true`. |
| `"use-components"` | Loads all components. Default is `true`. |
| `"use-utilities"` | Loads all utilities. Default is `true`. |
| `"use-icon"` | Loads the icons module. Default is `true`. |
