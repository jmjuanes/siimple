# About helpers

The **helpers** are a collection of utility-first and reusable CSS classes that you can use to build complex UI components or change the style of something. 

In a tradditional way, for creating a new component we should write the CSS styles in a `<style>` tag or in a separate CSS file:

```css
.card {
    background-color: #ffffff;
    border-radius: 0.75rem;
    display: flex;
    margin-left: auto;
    margin-right: auto;
    max-width: 30rem;
    padding: 1.5rem;
}
.card-image {
    flex-shrink: 0;
    margin-right: 1.5rem;
}
.card-image img {
    border-radius: 50%;
    height: 6rem;
    width: 6rem;
}
.card-content {
    flex-grow: 1;
}
.card-content-title {
    font-size: 2rem;
    font-weight: bold;
    margin-bottom: 0.5rem;
}
.card-content-description {
    color: rgba(116,124,139, 1);
}
```

And then use the previous styles in your HTML code as follows: 

```html
<div class="card">
    <div class="card-image">
        <img src="/landscape1.jpg" />
    </div>
    <div class="card-content">
        <div class="card-content-title">John Doe</div>
        <div class="card-content-description">
            Lorem ipsum dolor sit amet . . .
        </div>
    </div>
</div>
```

But using our **helpers** approach, you can just build this component using only HTML, without writting a single line of CSS and worring about classnames and reusability:

```html live=true bg=blue-200
<div class="has-bg-white has-radius-lg is-flex has-maxw-120 has-mx-auto has-p-6">
    <div class="has-flex-no-shrink has-mr-6">
        <img class="has-radius-full has-h-24 has-w-24" src="/landscape1.jpg" />
    </div>
    <div class="has-flex-grow">
        <div class="has-mb-2 has-weight-bold has-size-4">John Doe</div>
        <div class="has-text-gray-500">
            Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod 
            tempor incididunt ut labore et dolore magna aliqua.
        </div>
    </div>
</div>
```

## Using prebuilt helpers

We provide a collection of prebuilt helpers as a plugin. Just import the plugins module and add `plugins.helpers()` to the `$plugins` section of your configuration file:

```scss
@use "siimple/plugins" as plugins;

@use "siimple" with (
    $plugins: (
        plugins.helpers(),
    ),
);
```


## Creating new helpers

You can generate your own helpers using the `create-helper-plugin` function of the `siimple/lib` module. This function accepts only a map as an argument, where the configuration of the helper must be provided:

```scss
// my-helper.scss
@use "siimple/lib" as lib;

@function my-helper() {
    @return lib.create-helper-plugin((
        // helper configuration here
    ));
}
```

The configuration accepts the following fields:

- `name`: name of the new helper.
- `properties`: a list of CSS properties to set in the helper.
- `values`: a list of values to set in each CSS property specified in the `properties` field.
- `responsive`: a boolean to generate responsive variants of this helper.
- `states`: a list of states to generate variants. Allowed values: `default`, `hover` and `focus`.

The following code shows a simple example of a custom helper that will generate helpers to set the `background-color` property:

```scss
// my-helpers.scss
@use "siimple/lib" as lib;

@function background-colors() {
    @return lib.create-helper-plugin((
        "name": "bg",
        "properties": ("background-color"),
        "states": ("default" "hover"),
        "responsive": false,
        "values": (
            "transparent": transparent,
            "black": #000000,
            "white": #ffffff,
        ),
    ));
}
```

Now you can add your new helper plugin to the `$plugins` section of the siimple configuration:

```scss
@use "siimple/plugins" as plugins;
@use "./my-helpers.scss" as my-helpers;

@use "siimple" with (
    $plugins: (
        my-helpers.background-colors(),
    ),
));
```
